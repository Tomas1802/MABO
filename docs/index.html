Documentacion de usuario

Aprende MABO desde cero

MABO es un lenguaje de consola en espanol para administrar archivos, carpetas y automatizaciones con comandos faciles de leer.

Empezar con comandos Ver referencia

1. Instalacion en Windows

Instala MABO desde la ultima release de GitHub. El instalador descarga la distribucion, la copia en %LOCALAPPDATA%\MABO y agrega mabo al PATH de tu usuario.

Instalar

powershell -NoProfile -ExecutionPolicy Bypass -Command "irm https://github.com/Tomas1802/MABO/releases/latest/download/install-windows.ps1 | iex"

Abrir una nueva terminal

Cierra y abre PowerShell para que Windows cargue el PATH actualizado.

mabo

Desinstalar

powershell -NoProfile -ExecutionPolicy Bypass -Command "irm https://github.com/Tomas1802/MABO/releases/latest/download/uninstall-windows.ps1 | iex"
Ubicacion instalada: %LOCALAPPDATA%\MABO. El ejecutable queda en %LOCALAPPDATA%\MABO\bin\mabo.bat.
Logs de tareas: MABO muestra la ruta al iniciar. En Windows normalmente es %LOCALAPPDATA%\MABO\logs\tasks.

2. Formas de usar MABO

Modo interactivo

Es la forma recomendada para aprender. Escribes un comando, presionas Enter y ves el resultado.

mabo
mabo> Mostrar "Hola"
mabo> salir

Ejecutar un archivo

Guarda comandos en un archivo .mabo y ejecutalos como rutina.

mabo scripts\limpieza.mabo

Comando corto

Sirve para instrucciones simples. En Windows evita comillas internas complejas.

mabo -c "Mostrar 123"

Ayuda dentro del modo interactivo

mabo> ayuda
mabo> salir

3. Archivos .mabo

Los scripts propios de MABO usan la extension .mabo. Un archivo puede contener comandos sueltos, ciclos, condiciones, tareas o llamadas a PowerShell/Linux.

mabo examples/limpieza.mabo
mabo --check examples/limpieza.mabo

mabo archivo.mabo ejecuta el archivo. mabo --check archivo.mabo analiza sintaxis y semantica con ANTLR y SemanticVisitor, pero no ejecuta acciones.

Ejecutar archivos desde MABO

Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo"

Esta instruccion ejecuta el archivo inmediatamente. Para programarlo, agrega una programacion al final.

Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo" A Las "23:00"
Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo" Cada 24 Horas
Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo" Al Iniciar Sistema
Al Iniciar Sistema solo registra el arranque; no ejecuta el archivo en ese momento. A Las y Cada si quedan activos en el proceso actual.

Validar antes de ejecutar

mabo --check "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo"
mabo validar "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo"

Usa validacion cuando estes editando un archivo y quieras detectar errores sin mover, borrar o crear nada.

Extension simple para VS Code

El proyecto incluye una extension basica en vscode/mabo-language con resaltado de sintaxis, comentarios, pares de simbolos y comandos para ejecutar o validar el archivo actual.

cd vscode/mabo-language
code .

Desde VS Code puedes abrir esa carpeta y presionar F5 para probar la extension en una ventana de desarrollo. La extension espera que el comando mabo este instalado en PATH.

5. Rutas y seguridad

MABO trabaja con rutas del sistema de archivos. Por seguridad, valida que las rutas esten dentro de raices permitidas y bloquea rutas criticas del sistema.

Rutas recomendadas

Variable base = "%USERPROFILE%/Documents/MABO"
Variable documentos = base + "/Documentos"
Variable descargas = base + "/Descargas"

En los ejemplos usamos base como carpeta principal de trabajo. Puedes usar otro nombre, como proyecto o raiz, siempre que lo hayas creado con Variable nombre = valor.

Variables de entorno soportadas

FormatoEjemploUso
Windows%USERPROFILE%Carpeta del usuario actual
Windows%APPDATA%Configuracion de aplicaciones
Unix/Linux${HOME}Carpeta del usuario
Unix/Linux~Carpeta del usuario
MABO bloquea rutas raiz, System32, Program Files y otras ubicaciones peligrosas. Para trabajar sin problemas, usa una carpeta de proyecto dentro de Documents.

6. Navegacion

MABO tiene una carpeta actual. Las rutas relativas se resuelven desde esa carpeta y el modo interactivo la muestra en el prompt.

Mostrar Ruta
Ir A "%USERPROFILE%/Documents"
Ir A "C:\Users\tparr\Documents\TOMAS"
Ir A "TOMAS"
Ir A ".."
Crear Carpeta "Prueba"
Listar Contenido

Ir A cambia la carpeta actual de MABO. Mostrar Ruta imprime la carpeta actual en la consola. En Windows puedes escribir rutas con / o con \.

7. Variables

Las variables guardan valores para reutilizarlos. Se crean con Variable nombre = valor y se modifican con =.

Variable cantidad = 10
Variable nombre = "reporte.txt"
Variable activo = Verdadero
Variable colores = ["rojo", "verde", "azul"]
cantidad = cantidad + 1
Mostrar cantidad

Tipos comunes

TipoEjemploDescripcion
Numero42, 3.5Valores para calculos.
Texto"Hola"Cadenas entre comillas dobles.
BooleanoVerdadero, FalsoValores para condiciones.
Lista["a", "b"]Valores que puedes recorrer con Para Cada.

8. Mostrar resultados

Usa Mostrar para imprimir texto o valores en la consola.

Mostrar "Hola"
Mostrar 10
Mostrar Verdadero
Mostrar "Total: " + 25

El operador + concatena texto o suma numeros segun el contexto.

Mostrar 10 + 5
Mostrar "Hola " + "Mundo"
Mostrar base + "/archivo.txt"

9. Comandos para archivos

Crear Archivo

Crear Archivo "ruta/archivo.txt"

Crea un archivo vacio. Si la carpeta padre no existe, MABO intenta crearla.

Leer Archivo

Leer Archivo "ruta/archivo.txt"

Muestra el contenido y guarda el texto en ultimoResultado.

Escribir

Crea o reemplaza el contenido completo del archivo.

Anexar

Anexar "\nlinea nueva" En "ruta/archivo.txt"

Agrega texto al final sin borrar lo anterior.

Copiar Archivo

Copiar Archivo "origen.txt" A "destino.txt"

Copia el archivo. Si el destino existe puede pedir confirmacion.

Mover Archivo

Mover Archivo "origen.txt" A "carpeta/destino.txt"

Mueve el archivo a una nueva ruta.

Renombrar Archivo

Renombrar Archivo "viejo.txt" A "nuevo.txt"

Usa una nueva ruta con el nuevo nombre.

Eliminar Archivo

Eliminar Archivo "ruta/archivo.txt"
Eliminar Archivo "ruta/archivo.txt" Sin Confirmar

Elimina un archivo. Para practicar, usa primero Simular. Sin Confirmar omite la pregunta interactiva.

11. Busqueda y filtros

Los comandos de busqueda recorren una carpeta y devuelven rutas que coinciden con filtros. El resultado queda en ultimoResultado.

Buscar archivos

Buscar Archivos En "ruta" Con Extension ".pdf"
Buscar Archivos En "ruta" Con Nombre "factura"
Buscar Archivos En "ruta" Con Prefijo "backup"

En Mas Antiguos Que y Mas Recientes Que, la palabra Mas forma parte del filtro. No significa suma ni concatenacion.

Combinar filtros

Buscar carpetas

Buscar Carpetas En "ruta" Con Nombre "docs"
Los filtros de tamano siempre deben indicar unidad: Bytes, KB, MB o GB.

14. Funciones

Una funcion calcula o ejecuta algo reutilizable. Puede recibir parametros y devolver un valor con Retornar.

Funcion NombreCompleto(nombre, apellido)
    Retornar nombre + " " + apellido
FinFuncion

Variable persona = NombreCompleto("Ana", "Lopez")
Mostrar persona

Funcion que usa archivos

Funcion ContarTxt(ruta)
    Buscar Archivos En ruta Con Extension ".txt"
    Obtener Cantidad Archivos En ultimoResultado
    Retornar ultimoResultado
FinFuncion

15. Tareas

Una tarea es una rutina con nombre. No devuelve valor; sirve para agrupar pasos de automatizacion.

Tarea PrepararProyecto:
    Crear Carpeta base + "/entrada"
    Crear Carpeta base + "/salida"
    Mostrar "Proyecto preparado"
Fin

Ejecutar Tarea PrepararProyecto

Dentro del modo interactivo, MABO espera hasta que escribas Fin antes de ejecutar o registrar la tarea.

Las tareas y los archivos .mabo pueden ejecutarse inmediatamente, cada cierto tiempo, al iniciar el sistema o una sola vez a una hora especifica.

Programar tareas con nombre

Ejecutar Limpieza Cada 30 Minutos
Ejecutar Tarea Limpieza Cada 30 Minutos
Ejecutar Respaldo Cada 24 Horas
Ejecutar Respaldo Cada 1 Dias
Ejecutar Limpieza Al Iniciar Sistema
Ejecutar Respaldo A Las "23:00"
Ejecutar Tarea Respaldo A Las "23:00"

La palabra Tarea es opcional al ejecutar o programar tareas.

Programar archivos .mabo

Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo"
Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo" A Las "23:00"
Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo" Cada 24 Horas
Ejecutar Archivo "C:\Users\tparr\Documents\TAREAS\LimpiarXML.mabo" Al Iniciar Sistema

Programar el archivo completo suele ser la forma mas practica para rutinas de arranque, porque MABO guarda la ruta del .mabo y puede ejecutarlo despues de reiniciar.

Como se comporta cada programacion

ProgramacionComportamiento
A Las "HH:mm"Ejecuta una vez a la proxima ocurrencia de esa hora. Si termina correctamente, MABO la elimina.
Cada X Minutos/Horas/DiasEjecuta inmediatamente y luego repite mientras el proceso siga activo. La programacion queda guardada para restaurarla.
Al Iniciar SistemaRegistra el arranque del sistema. No ejecuta inmediatamente.

Las programaciones se guardan en %USERPROFILE%/Documents/MABO/schedules.ndjson.

Si vienes de una version anterior, MABO puede copiar programaciones antiguas a la nueva carpeta cuando la nueva ruta todavia no existe.

Administrar programaciones

Listar Tareas Programadas
Eliminar Tarea Programada Limpieza
Eliminar Tareas Programadas
Cambiar Programacion De Tarea Limpieza Cada 2 Horas
Cambiar Programacion De Tarea Limpieza A Las "23:30"

Listar Tareas Programadas muestra tareas y archivos programados. Eliminar Tarea Programada Nombre borra una programacion de tarea por nombre. Eliminar Tareas Programadas borra todas las programaciones, incluidas las de archivos .mabo y entradas de arranque creadas por MABO. Cambiar Programacion De Tarea reemplaza la programacion anterior de una tarea con nombre.

Las tareas programadas mantienen el proceso activo cuando usan el scheduler interno. Para detenerlo, usa Ctrl+C en la terminal.

17. Logs de tareas

Las tareas manuales y programadas escriben logs para que puedas revisar que hicieron cuando no estabas viendo la terminal.

%LOCALAPPDATA%\MABO\logs\tasks

Cada tarea crea un archivo por dia con este formato:

LimpiarArchivosXML-2026-05-31.log

Ese archivo incluye inicio, fin, errores, mensajes de Mostrar, busquedas, listados y operaciones como crear, escribir, copiar, mover, eliminar, comprimir y comandos nativos.

La misma carpeta contiene scheduler.log, que registra cuando se programa o restaura una tarea y si una ejecucion programada falla.

Las programaciones activas se guardan aparte en %USERPROFILE%/Documents/MABO/schedules.ndjson.

En Windows, Al Iniciar Sistema intenta usar Task Scheduler. Si Windows responde Acceso denegado, usa la carpeta Startup del usuario como respaldo. En ambos casos crea un wrapper en %LOCALAPPDATA%\MABO\startup y guarda su salida en %LOCALAPPDATA%\MABO\logs\startup.

%LOCALAPPDATA%\MABO\startup
%LOCALAPPDATA%\MABO\logs\startup
%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup
Si una tarea programada falla, MABO guarda el error. Las tareas de intervalo siguen programadas; las tareas de una sola ejecucion quedan registradas para diagnostico hasta que las elimines o cambies.

18. Modo simulacion

Simular ejecuta la validacion y auditoria, pero no modifica archivos ni ejecuta comandos nativos reales.

Simular Eliminar Archivo base + "/temp/cache.tmp"
Simular Copiar Carpeta base + "/datos" A base + "/backup"
Simular Comprimir base + "/docs" A base + "/docs.zip"
Simular Ejecutar PowerShell "Write-Output 'prueba'"

Usalo antes de acciones peligrosas, especialmente eliminaciones y comandos nativos.

19. Manejo de errores

Usa Intentar y Capturar para continuar cuando algo falle.

Intentar
    Leer Archivo base + "/no_existe.txt"
    Mostrar "Esto no se mostrara si falla"
Capturar
    Mostrar "No se pudo leer el archivo, continuo"
FinIntentar

20. PowerShell y Linux

Cuando MABO no tenga un comando propio para algo, puedes delegar a PowerShell en Windows o a un shell Linux en Linux/Unix.

Ejecutar PowerShell "Get-ChildItem"
Ejecutar Linux "ls -la"

Reglas de seguridad

  • PowerShell real solo se ejecuta automaticamente en Windows.
  • Linux real solo se ejecuta automaticamente en Linux/Unix.
  • La simulacion funciona para ambos sin ejecutar nada.
  • Patrones peligrosos como borrados masivos se bloquean.
  • Comandos potencialmente destructivos pueden pedir confirmacion.

Resultado

El resultado queda en ultimoResultado como un objeto con shell, comando, codigo de salida, stdout, stderr y bandera de simulacion.

21. Backup y compresion

Crear Backup De base + "/datos" En base + "/respaldo/datos"
Comprimir base + "/datos" A base + "/respaldo/datos_zip"
Descomprimir base + "/respaldo/datos_zip.zip" En base + "/restaurado"

Si el destino de Comprimir no termina en .zip, MABO agrega la extension automaticamente.

22. Permisos

En sistemas POSIX puedes cambiar permisos con formato octal o simbolico.

Cambiar Permisos "scripts/run.sh" A "755"
Cambiar Permisos "datos.txt" A "rw-r-----"
En Windows esta operacion muestra un error claro porque los permisos POSIX no estan disponibles.

25. Problemas comunes

El nombre de mi tarea no funciona

No uses palabras reservadas como nombre de tarea o variable. Por ejemplo, Backup es una palabra del lenguaje. Usa BackupDiario o RespaldoDiario.

Me sale acceso denegado

MABO permite trabajar dentro de tu perfil de usuario y dentro de la carpeta actual del proyecto. Tambien puedes guardar una ruta segura con Variable nombre = valor para registrarla como raiz permitida. Evita rutas del sistema.

El comando con -c falla por comillas

En Windows, usa el modo interactivo para comandos con textos o rutas entre comillas. -c funciona mejor con instrucciones simples.

Una tarea programada deja la terminal abierta

Es normal cuando usa el scheduler interno. Presiona Ctrl+C para detenerlo.

No se que hizo una tarea programada

Revisa %LOCALAPPDATA%\MABO\logs\tasks. Cada tarea tiene un archivo por dia y el scheduler usa scheduler.log.
